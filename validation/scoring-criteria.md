# 棋牌类游戏质量验收标准

## 1. 基础质量门禁

- 代码质量：`lint + 单元测试 + 集成测试` 全部通过。
- 安全质量：依赖漏洞扫描和镜像扫描无高危阻断项。
- 性能质量：`P95 <= 200ms`。
- 体验质量：连续 `3` 次评估分数 `>= 9.5`。
- 可交付性：文档、配置、流程定义齐全。

## 2. UI/UX 截图验收（必选）

- 必须同时提交桌面端和移动端截图。
- 截图总数不少于 `10` 张。
- 必须覆盖以下场景：
1. 大厅首页（lobby-home）
2. 房间列表（room-list）
3. 对局中牌桌/棋盘（table-in-game）
4. 手牌/棋盘状态区（hand-or-board-status）
5. 结算页（settlement-result）
6. 断线重连页（reconnect-state）
7. 余额不足页（insufficient-balance）
8. 网络延迟告警（network-latency-warning）
9. 新手引导（new-user-onboarding）
10. 无障碍高对比模式（accessibility-high-contrast）

## 3. UI/UX 评分规则（满分 10）

- 总分公式：`总分 = UI完整性*40% + UI设计精美*30% + UX设计合理*30%`
- UI 完整性（硬门槛 `>= 9.5`）
1. 核心页面元素无缺失
2. 棋盘/牌桌比例正确且无遮挡
3. 玩家位、计时器、分数、操作区完整
4. 空状态/加载态/错误态可见且可用
- UI 设计精美（硬门槛 `>= 9.3`）
1. 视觉层级清晰，主次分明
2. 棋牌元素风格统一，质感稳定
3. 字体、间距、色彩在多页面一致
4. 复杂背景下文字和按钮对比度充足
- UX 设计合理（硬门槛 `>= 9.3`）
1. 核心路径步数最短且无歧义
2. 关键操作有即时反馈与可撤销策略
3. 断线重连、托管、超时等异常流程可闭环
4. 移动端触控热区和手势冲突可控

## 4. 一票否决项

- 对局主界面关键操作按钮缺失。
- 棋盘/牌桌或手牌区出现错位导致不可用。
- 断线重连后状态不一致。
- 文字不可读（对比度严重不足）或移动端核心操作点击困难。
